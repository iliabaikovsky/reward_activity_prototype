# Работа с Figma в Cursor (MCP)

Цель: ты кидаешь **ссылку на Figma**, агент подключается к макету и пишет код в этом репозитории (React + Vite), потом проект деплоится на Vercel.

## Что нужно с твоей стороны

1. Проект открыт как **корень скопированного шаблона**, и ты уже сделал **`npm install`** там (не обязательно в репозитории workshop).
2. **Figma MCP** включён в Cursor и ты залогинен в Figma (как в документации Cursor / Figma).
3. В **`DESIGN.md`** актуальная ссылка и краткое ТЗ (экраны, вьюпорт).
4. Доступ к файлу в Figma у аккаунта, с которым работает MCP.

## Что написать агенту (примеры)

- «Ссылка на Figma: `https://figma.com/design/...`. Сверстай фрейм “Checkout” как в макете, mobile 375. Токены вынеси в `:root` в `styles.css`.»
- «По этой же ссылке добавь второй экран и переход по кнопке.»
- «Сверь отступы и типографику со скриншотом ноды из Figma.»

## Как агент использует Figma

- Из URL берутся **fileKey** и **node-id** (`1-2` в URL → node id `1:2`).
- Инструмент дизайн-контекста возвращает **референс-код** и **скриншот** — это ориентир, не копипаста: нужно **привести к структуре этого шаблона** (компоненты в `src/`, стили в проекте).

## Чеклист качества прототипа

- [ ] Вьюпорт / max-width как в макете или в `DESIGN.md`
- [ ] Шрифты подключены (Google Fonts или файлы)
- [ ] Интерактив: клики, фокус, базовые состояния кнопок
- [ ] `npm run build` проходит без ошибок
- [ ] После деплоя проверить на телефоне (Vercel URL)

## Деплой

См. **`DEPLOY.md`**. Первый раз: `npx vercel login`, затем из папки проекта `npm run deploy`.
